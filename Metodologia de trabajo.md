# Metodología de Trabajo para Equipo de Desarrollo

## 1. Objetivo

Establecer un proceso de trabajo simple para organizar el desarrollo del proyecto, mejorar la comunicación del equipo y dar seguimiento a los avances mediante reuniones semanales (weeklies).

---

# 2. Estructura del equipo

## Líder de Proyecto
Responsabilidades:
- Definir prioridades del proyecto
- Coordinar al equipo
- Revisar avances
- Resolver bloqueos

## Desarrolladores
Responsabilidades:
- Implementar funcionalidades
- Corregir errores
- Participar en reuniones semanales
- Subir código al repositorio


---

# 3. Organización del trabajo

El trabajo se organiza en un **Backlog del proyecto**, que es una lista de todas las tareas pendientes.

Cada tarea debe incluir:

- Descripción
- Responsable
- Estado

## Estados de las tareas

- Pendiente
- En progreso
- En revisión
- Terminado

## Ejemplo de tareas

- Crear sistema de login
- Implementar base de datos de usuarios
- Diseñar interfaz de registro
- Implementar carrito de compras

---

# 4. Ciclo de trabajo semanal

El equipo trabaja en ciclos de **una semana**.

Cada semana sigue el siguiente proceso.

---

# 5. Planeación semanal

Duración aproximada: **30–45 minutos**

Se realiza al inicio de la semana.

Objetivos:

- Revisar el backlog
- Seleccionar tareas para la semana
- Asignar responsables

Preguntas clave:

- ¿Qué tareas son prioritarias?
- ¿Quién trabajará en cada tarea?
- ¿Existen dependencias entre tareas?

Resultado:

Lista de tareas comprometidas para esa semana.

---

# 6. Desarrollo durante la semana

Cada desarrollador trabaja en sus tareas asignadas.

Buenas prácticas:

- Subir cambios frecuentemente al repositorio
- Probar las funcionalidades antes de marcarlas como terminadas
- Mantener comunicación con el equipo


---

# 7. Reunión Weekly

Duración recomendada: **15–20 minutos**

Frecuencia: **una vez por semana**

Cada miembro responde tres preguntas:

1. ¿Qué hice la semana pasada?
2. ¿Qué voy a hacer esta semana?
3. ¿Tengo algún bloqueo o problema?

Ejemplo:

Desarrollador A:

- Terminé el módulo de registro
- Esta semana trabajaré en el login
- Necesito acceso a la base de datos

Objetivo de la reunión:

- Detectar problemas rápidamente
- Coordinar el trabajo del equipo
- Ajustar prioridades

---

# 8. Revisión de avances

Al final de la semana se revisa:

- Qué tareas se completaron
- Qué tareas quedaron pendientes
- Qué problemas surgieron

Opcionalmente se puede realizar una demostración del avance del sistema.

---

# 9. Mejora continua

El equipo analiza brevemente:

- Qué funcionó bien
- Qué se puede mejorar

Ejemplo:

Problema:
Las tareas eran demasiado grandes.

Solución:
Dividir tareas en partes más pequeñas.

---

# 10. Flujo de la metodología

Backlog del proyecto  
↓  
Planeación semanal  
↓  
Desarrollo durante la semana  
↓  
Reunión Weekly  
↓  
Revisión de avances  
↓  
Mejora continua
